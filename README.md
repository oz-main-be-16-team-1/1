# Django 프로젝트 1팀

![ERD 구조](./project_erd.png)


- 요구사항
    
    
    | 테이블 이름 | 설명 | 필드 |  |
    | --- | --- | --- | --- |
    | **users** | 유저 모델을 저장할 테이블입니다. | - 이메일 (로그인 시 사용) <br>- 비밀번호 <br>- 닉네임<br>- 이름 <br>- 전화번호 <br>- 마지막 로그인 <br>- 스태프 여부 <br>- 관리자 여부 <br>- 계정 활성화 여부 <br>- 생성일시 |  |
    | **accounts** | 유저가 보유한 계좌 정보를 저장할 테이블입니다. | - 유저 정보 <br>- 은행 코드 (카카오뱅크, KB, NH, IBK 등) <br>- 계좌번호 <br>- 계좌유형 (입출금,저축통장,정기예금,정기적금) <br>- 계좌잔액 <br>- 생성일시 |  |
    | **transactions** | 사용자가 보유한 계좌의 거래 내역을 저장할 테이블입니다. | - 계좌 정보 <br>- 거래 금액 <br>- 거래 내역 (거래 명세서) <br>- 입출금 타입 (입금, 출금) <br>- 거래 타입 (현금, 계좌, 자동 이체,카드결제 <br>- 거래 후 잔액<br>- 거래 일시 |  |
    





🔗 데이터베이스 관계 정의 (Relationships)


users → accounts <br>	· user_id	1 : N	한 명의 사용자는 여러 개의 계좌를 보유할 수 있습니다.

accounts → transactions	<br> · account_id	1 : N	하나의 계좌에서는 여러 건의 거래 내역이 발생할 수 있습니다.
